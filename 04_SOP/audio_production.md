## 1. 音频制作流程总览

音频生产按以下五个阶段顺序推进，每阶段产出需通过质量检查后才能进入下一阶段。

| 阶段 | 工具 / 平台 | 产出 | 责任人 |
|------|------------|------|--------|
| ① 配音生成 | ElevenLabs API / 网页 | 9 条对白 MP3 → 转 WAV | 音频工程师 |
| ② 配乐生成 | Suno / Udio | 3 段配乐 MP3/WAV | 音乐设计 |
| ③ 音效设计 | ElevenLabs SFX / AudioLDM / 素材库 | 环境、动作、特殊音效素材 | 音效设计 |
| ④ 混音 | DaVinci Resolve Fairlight / 专业 DAW | 多轨混音工程 + 立体声混音 | 混音师 |
| ⑤ 母版 | DaVinci Resolve / 母带工具 | 嵌入视频 AAC + 独立 WAV 母版 | 混音师 |

### 1.1 流程图

```
配音生成 ──┐
配乐生成 ──┼──→ 音效设计 ──→ 混音（对白/配乐/音效分轨）──→ 母版输出 ──→ 归档
           │
           └─ 三者可并行启动，最终在混音阶段汇合
```

### 1.2 关键时间节点（建议）

| 节点 | 工作日 | 内容 |
|------|--------|------|
| D0 | 第 1 天 | 配音 + 配乐生成启动（并行） |
| D1 | 第 2 天 | 音效素材采集与生成 |
| D2 | 第 3 天 | 对白处理（EQ/压缩/混响） |
| D3 | 第 4 天 | 多轨混音初版 |
| D4 | 第 5 天 | 混音修订 + 母版输出 |
| D5 | 第 6 天 | 质量检查与归档 |

### 2.5 对白处理链

对白生成后，在 DAW 中按以下链路处理：

```
原始 WAV → 降噪 → EQ → 压缩 → 去齿音 → 混响/空间 → 响度归一
```

| 处理环节 | 艾娃 | 核心 |
|---------|------|------|
| 降噪 | 轻度（保留呼吸感） | 中度（去除环境底噪，突出合成质感） |
| EQ | 高通 80 Hz，2–4 kHz 提升 1–2 dB 增加临场感 | 高通 100 Hz，5–8 kHz 衰减 2 dB 弱化齿音，加 200 Hz 轻微衰减增强"远距感" |
| 压缩 | 比例 3:1，阈值 -18 dB，attack 10 ms，release 100 ms | 比例 4:1，阈值 -20 dB，attack 5 ms，release 80 ms（更"贴脸"） |
| 去齿音 | 阈值 -20 dB，频率 6 kHz | 阈值 -18 dB，频率 7 kHz |
| 混响 | 不加或极轻房间混响（< 0.5 s），保持干爽 | 大空间混响 2–3 s，预延迟 30 ms，配合轻微 chorus/delay 模拟电子远距感 |
| 响度归一 | -12 LUFS | -12 LUFS（混响后整体） |

### 2.6 核心声音电子混响配方

核心声音的"远处传来 + 电子混响"质感是本片关键听感，建议在 DAW 中叠加：

1. **Plate Reverb**：decay 2.5 s，size 80%，damping 40%
2. **Stereo Delay**：左 1/8 音符 250 ms，右 3/8 音符 375 ms，反馈 20%，混合 15%
3. **Chorus**：rate 0.3 Hz，depth 30%，mix 20%
4. **高通滤波**：在效果链末端加 150 Hz 高通，避免低频混浊

## 3. 配乐规范（Suno/Udio）

### 3.1 配乐需求总览

全片配乐分 3 段，对应叙事三幕结构。每段生成后需裁剪、淡入淡出、音量平衡，再进入混音阶段。

| 序号 | 标题 | 对应段落 | 时长 | 情绪 |
|------|------|---------|------|------|
| M1 | Ruins Dawn（废墟氛围） | 开场—艾娃进入废墟（0:00–1:30） | 120 s | 孤寂、辽阔、废墟晨光 |
| M2 | Core Awakening（核心觉醒） | 核心首次发声—对峙（1:30–3:30） | 120 s | 神秘、神圣、张力上升 |
| M3 | Echo Resolution（结尾释然） | 释然—结尾（3:30–5:00） | 120 s | 释然、希望、新开始 |

### 3.2 各段配乐详细规格

#### M1 — Ruins Dawn（废墟氛围）

| 项目 | 内容 |
|------|------|
| 风格标签（tags） | `sci-fi, cinematic, ambient, electronic, lonely, vast` |
| 提示词（prompt） | A slow, atmospheric sci-fi ambient track. Sparse piano notes, distant synth pads, low rumble, wind-like textures. Evokes a lone wanderer in a vast ruined city at dawn. |
| 时长 | 120 s |
| 情绪曲线 | 0–30 s 极简铺底（低频 + 风声质感）→ 30–60 s 加入稀疏钢琴 → 60–90 s 合成 pad 渐强 → 90–120 s 回落留白，为对白让路 |
| make_instrumental | `true` |

#### M2 — Core Awakening（核心觉醒）

| 项目 | 内容 |
|------|------|
| 风格标签（tags） | `sci-fi, cinematic, orchestral electronic, mysterious, sacred` |
| 提示词（prompt） | Mysterious and sacred sci-fi music. Choir-like synths, pulsating electronic arpeggios, building tension. Sense of discovering an ancient alien AI core. |
| 时长 | 120 s |
| 情绪曲线 | 0–20 s 低频脉冲 + 合唱 pad（神秘）→ 20–60 s 电子琶音渐入（觉醒）→ 60–100 s 张力上升、节奏加密（对峙）→ 100–120 s 突然抽空，留核心低频脉冲 |
| make_instrumental | `true` |

#### M3 — Echo Resolution（结尾释然）

| 项目 | 内容 |
|------|------|
| 风格标签（tags） | `sci-fi, cinematic, emotional, hopeful, strings` |
| 提示词（prompt） | Emotional sci-fi cinematic music. Warm strings, gentle piano, swelling synths. A feeling of release, acceptance, and a new beginning after a long silence. |
| 时长 | 120 s |
| 情绪曲线 | 0–30 s 温暖弦乐 + 钢琴（释然）→ 30–70 s 合成渐强、情绪上扬（希望）→ 70–100 s 高潮（新开始）→ 100–120 s 缓慢淡出，留余韵 |
| make_instrumental | `true` |

### 3.3 Suno 调用参数

调用脚本见 `08_Automation/suno_music_api.py`。核心 payload 字段：

```json
{
  "title": "Ruins Dawn",
  "tags": "sci-fi, cinematic, ambient, electronic, lonely, vast",
  "prompt": "A slow, atmospheric sci-fi ambient track...",
  "duration": 120,
  "make_instrumental": true
}
```

| 参数 | 取值 | 说明 |
|------|------|------|
| `make_instrumental` | `true` | 全部配乐为纯器乐，避免人声干扰对白 |
| `tags` | 见各段 | 风格标签，影响生成方向 |
| `prompt` | 见各段 | 自然语言描述，建议英文以获得更稳定结果 |
| `duration` | `120` | Suno 单次生成上限约 4 分钟，120 s 足够覆盖单段 |

### 3.4 Udio 备选方案

若 Suno 生成结果不理想，可使用 Udio 作为备选：

- Udio 在管弦乐与情感表达上常优于 Suno，适合 M2、M3
- Udio 支持扩展（Extend）功能，可在已有片段基础上续写
- 同样使用 `make_instrumental` 等效选项

### 3.5 输出后处理

每段配乐生成后，在 DAW 中执行以下后处理：

1. **裁剪**：按情绪曲线与画面时间码裁剪至实际需要长度（通常 60–90 s）
2. **淡入淡出**：淡入 2 s，淡出 3 s（结尾段淡出可延长至 5 s）
3. **音量平衡**：将配乐整体响度归一至 -20 LUFS（混音阶段再细调）
4. **频谱避让**：在 200–500 Hz 与 2–4 kHz 做轻度侧链衰减，为对白让出空间
5. **多版本归档**：保留 Suno 原始版本（`_v1`、`_v2`）以便回退

## 4. 音效规范

### 4.1 环境音分类

| 类别 | 描述 | 出现场景 | 来源 |
|------|------|---------|------|

| 机械嗡鸣 | 远处休眠设备的低频嗡鸣 | 核心舱附近 | ElevenLabs SFX / 合成 |

### 4.2 动作音效

| 动作 | 描述 | 触发时间点 | 来源 |
|------|------|-----------|------|
| 触碰金属 | 艾娃手触碰核心外壳的金属共振声 | 艾娃伸手镜头 | Foley / 素材库 |

### 4.3 特殊音效

| 音效 | 描述 | 使用场景 | 来源 |
|------|------|---------|------|
| 核心低频脉冲 | 40–60 Hz 周期性脉冲，像巨大机械的心跳 | 核心存在感提示，贯穿 M2 段 | 合成（正弦波 + LFO） |
| 数据流电子音 | 高频颗粒状电子音流，快速流动 | 核心传输信息 | AudioLDM / ElevenLabs SFX |
| 回响共鸣 | 核心发声时的空间共鸣尾音 | 核心每句对白结尾 | 混响处理（见 2.6） |

### 4.4 音效来源

| 来源 | 适用 | 备注 |
|------|------|------|
| ElevenLabs Sound Effects | 环境、动作、特殊音效 | 文本提示生成，快速迭代 |
| AudioLDM | 数据流、电子音 | 开源模型，可本地部署 |
| 素材库（Freesound / Epidemic Sound 等） | 风声、金属、脚步 | 注意授权合规，见 `06_Research/licensing_compliance.md` |
| Foley 录制 | 脚步、触碰金属 | 实地或棚录，质感最真实 |

### 4.5 音效层叠规范

混音时音效按四层叠加，每层独立音轨、独立响度控制：

```
┌─────────────────────────────┐
│ 特殊层  │ 核心脉冲/数据流/共鸣  │  -18 LUFS
├─────────────────────────────┤
│ 动作层  │ 触碰/舱门/电流        │  -18 LUFS
├─────────────────────────────┤
│ 环境层  │ 风声/沙尘/金属吱嘎    │  -24 LUFS
├─────────────────────────────┤
│ 底噪层  │ 机械嗡鸣/空间底噪     │  -30 LUFS
└─────────────────────────────┘
```

| 层级 | 内容 | 目标响度 | 说明 |
|------|------|---------|------|
| 底噪层 | 机械嗡鸣、空间底噪 | -30 LUFS | 全片持续，营造空间真实感 |
| 环境层 | 风声、沙尘、金属吱嘎 | -24 LUFS | 随场景变化，镜头切换时交叉淡变 |
| 动作层 | 触碰、舱门、电流 | -18 LUFS | 与画面动作严格同步 |
| 特殊层 | 核心脉冲、数据流、共鸣 | -18 LUFS | 核心相关镜头突出，其余时段衰减 |

## 5. 混音规范（DaVinci Resolve Fairlight / 专业 DAW）

### 5.1 音轨结构

混音工程按以下轨道组织，所有轨道汇入总线（Master Bus）。

| 轨道编号 | 轨道名 | 类型 | 内容 | 目标响度 |
|---------|--------|------|------|---------|
| T1 | 对白轨（Dialogue） | 立体声 | 9 条对白 | -12 LUFS |
| T2 | 配乐轨（Music） | 立体声 | 3 段配乐 | -20 LUFS |
| T3 | 音效-底噪层 | 立体声 | 机械嗡鸣、空间底噪 | -30 LUFS |
| T4 | 音效-环境层 | 立体声 | 风声、沙尘、金属吱嘎 | -24 LUFS |
| T5 | 音效-动作层 | 立体声 | 触碰、舱门、电流 | -18 LUFS |
| T6 | 音效-特殊层 | 立体声 | 核心脉冲、数据流、共鸣 | -18 LUFS |
| BUS | 总线（Master Bus） | 立体声 | 全部混音输出 | -14 LUFS |

### 5.2 响度标准

| 对象 | 目标响度 | 说明 |
|------|---------|------|
| 整体成片 | -14 LUFS | YouTube / B 站推荐标准 |
| 对白 | -12 LUFS | 对白优先，确保可懂度 |
| 配乐 | -20 LUFS | 衬托对白，不抢戏 |
| 音效（动作/特殊） | -18 LUFS | 增强临场感 |
| 音效（环境/底噪） | -24 / -30 LUFS | 铺底，不可喧宾夺主 |
| True Peak（整体） | ≤ -1 dB | 防止平台转码削波 |

### 5.3 EQ 建议

| 对象 | EQ 设置 | 目的 |
|------|---------|------|
| 对白轨 | 高通 80 Hz（12 dB/oct） | 去除低频隆隆声与口风噪 |
| 对白轨 | 2–4 kHz 提升 1–2 dB | 增强临场感与可懂度 |
| 配乐轨 | 低通 12 kHz（12 dB/oct） | 为对白让出高频空间 |
| 配乐轨 | 200–500 Hz 轻度衰减 2 dB | 避免与对白中频冲突 |
| 音效轨 | 按需高通 60 Hz | 去除不需要的低频 |

### 5.4 动态范围处理

| 对象 | 压缩比 | 阈值 | Attack / Release | 说明 |
|------|--------|------|------------------|------|
| 对白轨 | 3:1 | -18 dB | 10 ms / 100 ms | 控制动态，保持自然 |
| 配乐轨 | 2:1 | -22 dB | 30 ms / 200 ms | 轻度压缩，保留起伏 |
| 总线 | 2:1 | -6 dB | 50 ms / 300 ms | 总线限幅前预压缩 |
| 限幅器（总线末端） | — | True Peak -1 dB | — | 防止削波 |

### 5.5 空间处理

| 对象 | 空间处理 | 参数 |
|------|---------|------|
| 艾娃对白 | 保持干爽，不加或仅极轻房间混响 | 房间混响 < 0.5 s，混合 5% |
| 核心对白 | 大空间混响 + 电子延迟 | Reverb decay 2–3 s，预延迟 30 ms；详见 [2.6 核心声音电子混响配方](#26-核心声音电子混响配方) |
| 配乐 | 不额外加混响（生成时已有空间感） | — |
| 音效-环境层 | 轻度房间混响 | decay 1–1.5 s，混合 10% |
| 音效-特殊层 | 核心脉冲加长混响 | decay 3–4 s，混合 20% |

### 5.6 自动化（Automation）

| 参数 | 自动化场景 |
|------|-----------|
| 配乐音量 | 对白出现时配乐侧链衰减 3–6 dB（ducking），对白结束后 0.5 s 恢复 |
| 音效-环境层 | 镜头切换时交叉淡变 0.3–0.5 s |
| 核心混响量 | 核心对白结尾时混响尾音渐强，强化"远去"感 |
| 总线响度 | 片尾淡出 3 s 至 -∞ |

## 6. 母版输出规范

### 6.1 母版技术规格

| 项目 | 规格 |
|------|------|
| 采样率 | 48000 Hz |
| 位深 | 24 bit |
| 声道 | 立体声（Stereo） |
| 整体响度 | -14 LUFS（YouTube / B 站标准） |
| True Peak | ≤ -1 dB |
| 响度范围（LRA） | 建议 6–9 LU（保留动态） |

### 6.2 输出交付物

| 交付物 | 格式 | 用途 |
|--------|------|------|
| 嵌入视频的音频 | AAC，320 kbps，48 kHz | 与视频封装为 MP4/MOV，用于平台发布 |
| 独立 WAV 母版 | WAV，48 kHz，24 bit，立体声 | 归档与后期再加工 |
| 多轨工程文件 | DaVinci Resolve `.drp` / DAW 工程 | 归档，便于修订 |

### 6.3 母版检查流程

1. 在 DaVinci Resolve 中渲染整片，导出 AAC 320 kbps 嵌入视频
2. 单独导出 WAV 母版（48 kHz / 24 bit）
3. 使用响度表（如 Youlean Loudness Meter）复核整体 -14 LUFS、True Peak ≤ -1 dB
4. 在不同设备（监听音箱、耳机、笔记本扬声器、手机外放）上试听，确认对白可懂度
5. 归档至 `01_Assets/Audio/Master/`（见第 7 节命名规范）

## 7. 文件命名规范

### 7.1 总体原则

- 全部使用半角英文 + 下划线分隔，避免中文与空格
- 版本号用 `v1`、`v2`，最终版用 `final`
- 日期采用 `YYYYMMDD`

### 7.2 命名模板

| 类别 | 命名模板 | 示例 |
|------|---------|------|
| 对白 | `{角色}_{序号}.wav` | `Ava_01.wav`、`Core_05.wav` |
| 对白版本 | `{角色}_{序号}_v{版本}.wav` | `Ava_01_v2.wav` |
| 配乐 | `{曲名}_v{版本}.{ext}` | `RuinsDawn_v1.mp3`、`CoreAwakening_final.wav` |
| 音效 | `SFX_{类别}_{描述}_v{版本}.wav` | `SFX_Env_Wind_v1.wav`、`SFX_Action_DoorHydraulic_v1.wav`、`SFX_Special_CorePulse_v1.wav` |
| 混音工程 | `Singularity_Mix_{日期}.drp` | `Singularity_Mix_20260623.drp` |
| 母版（嵌入） | `Singularity_Master_{日期}.mp4` | `Singularity_Master_20260623.mp4` |
| 母版（独立音频） | `Singularity_MasterAudio_{日期}.wav` | `Singularity_MasterAudio_20260623.wav` |

### 7.3 目录结构

```
01_Assets/Audio/
├── Voices/          # 对白
│   ├── Ava_01.wav
│   ├── Core_01.wav
│   └── ...
├── Music/           # 配乐
│   ├── RuinsDawn_v1.mp3

│   ├── Env/         # 环境层

├── Mix/             # 混音工程
│   └── Singularity_Mix_20260623.drp
└── Master/          # 母版
    ├── Singularity_Master_20260623.mp4
    └── Singularity_MasterAudio_20260623.wav
```

## 8. 质量检查清单

混音与母版交付前，逐项核对以下清单。全部通过后方可归档发布。

### 8.1 配音检查

- [ ] 9 条对白全部生成，文件名与 `LINES` 列表一致
- [ ] 艾娃声线符合"低沉冷静、带疲惫"定位
- [ ] 核心声线符合"中性冷静、远处传来"定位
- [ ] 核心声音已加电子混响（2–3 s 大空间 + 延迟 + chorus）
- [ ] 艾娃声音保持干爽，无过度混响
- [ ] 对白无齿音爆音、无明显机械杂音
- [ ] 对白响度归一至 -12 LUFS

### 8.2 配乐检查

- [ ] 3 段配乐（Ruins Dawn / Core Awakening / Echo Resolution）全部生成
- [ ] 全部为纯器乐（`make_instrumental = true`）
- [ ] 已按时间码裁剪至实际需要长度
- [ ] 淡入淡出自然，无突变
- [ ] 配乐响度归一至 -20 LUFS
- [ ] 对白出现处配乐已做侧链衰减

### 8.3 音效检查

- [ ] 底噪层、环境层、动作层、特殊层四层齐全
- [ ] 动作音效与画面严格同步（帧级对齐）
- [ ] 核心低频脉冲贯穿 M2 段，节奏稳定
- [ ] 音效无削波、无数字爆音
- [ ] 音效来源均已确认授权合规

### 8.4 混音检查

- [ ] 音轨结构符合第 5.1 节规范
- [ ] 整体响度 -14 LUFS
- [ ] 对白 -12 LUFS，配乐 -20 LUFS，音效分层达标
- [ ] True Peak ≤ -1 dB
- [ ] 对白高通 80 Hz 已应用
- [ ] 配乐低通 12 kHz 已应用
- [ ] 对白压缩 3:1 / 阈值 -18 dB 已应用
- [ ] 对白在多设备（音箱/耳机/手机外放）上均可清晰听清
- [ ] 无相位问题（用相关表检查，立体声相关度 > 0）

### 8.5 母版检查

- [ ] 采样率 48 kHz、位深 24 bit、立体声
- [ ] 嵌入视频的 AAC 320 kbps 已导出
- [ ] 独立 WAV 母版已导出
- [ ] 多轨工程文件已归档
- [ ] 文件命名符合第 7 节规范
- [ ] 全片从头到尾完整试听一遍，无异常
