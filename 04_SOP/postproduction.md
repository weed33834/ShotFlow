## 一、后期制作流程总览

AI 生成视频片段本身存在分辨率低、闪烁、角色不一致、手指变形等问题，后期需经过多道工序才能输出可交付的 4K 母版。整体流程严格按以下顺序执行，每道工序产出独立的中间版本，便于回溯与返工。

### 1.1 流程图

```
原始片段 (Wan2.2 / 可灵 720P–1080P)
        │
        ▼
  ① 粗剪 (剪映专业版)        ── 按分镜表排列、定节奏、定结构
        │
        ▼
  ② 精剪 (DaVinci Resolve)   ── 逐帧修剪、转场、声画对位
        │
        ▼
  ③ 调色 (DaVinci Resolve)   ── Teal & Orange、ACES 工作流
        │
        ▼
  ④ 画质增强 (Topaz Video AI) ── 超分至 4K、降噪、锐化
        │
        ▼
  ⑤ 瑕疵修复 (After Effects) ── 闪烁、穿帮、手指、角色一致性
        │
        ▼
  ⑥ 混音 (DaVinci Fairlight) ── 对白、配乐、音效平衡、响度归一
        │
        ▼
  ⑦ 母版输出                 ── ProRes 422 HQ 4K 母版 + 发布衍生版
```

### 1.2 工序职责与产出

| 序号 | 工序 | 主用工具 | 责任人 | 输入 | 输出 | 存放位置 |
|------|------|----------|--------|------|------|----------|
| ① | 粗剪 | 剪映专业版 | 后期总监 | 原始片段 + 分镜表 | `roughcut_vNN.mp4` | `05_Output/Rough_Cuts/` |
| ② | 精剪 | DaVinci Resolve | 后期总监 | 粗剪版 + 配音/音效参考 | `finecut_vNN.mov`（ProRes 422 HQ） | `05_Output/Rough_Cuts/finecut/` |
| ③ | 调色 | DaVinci Resolve | 后期总监 | 精剪版 | `colorgraded_vNN.mov` | `05_Output/Final/` |
| ④ | 画质增强 | Topaz Video AI | 视频工程师 | 调色版（或原始片段逐镜） | `upscaled_4k_vNN.mov` | `05_Output/Final/` |
| ⑤ | 瑕疵修复 | After Effects | 合成师 | 增强版 | `repaired_vNN.mov` | `05_Output/Final/` |
| ⑥ | 混音 | DaVinci Fairlight | 音频工程师 | 修复版 + 配音 + 配乐 + 音效 | `mix_vNN.wav` / 内嵌版 | `01_Assets/Audio/` |
| ⑦ | 母版输出 | DaVinci Resolve | 后期总监 / PM | 混音版 | 母版 + 发布衍生版 | `05_Output/Final/` |

> **关键原则**：每道工序产出独立文件，**严禁覆盖上一工序产物**。返工时回到对应工序版本号递增，不破坏既有版本链。

### 1.3 工序顺序说明

- **调色在画质增强之前**：先在原始分辨率下完成调色，再交由 Topaz 超分，避免超分后再次调色引入的色彩偏移与算力浪费。若超分后出现局部偏色，可在修复阶段做微调。
- **画质增强在修复之前**：超分与降噪先行，修复阶段在干净的 4K 画面上做逐帧修补，效率更高。
- **混音在最后**：画面定稿后再做混音，避免画面改动导致声画错位。

## 二、剪辑规范

### 2.1 工具分工

| 阶段 | 工具 | 用途 | 说明 |
|------|------|------|------|
| 粗剪 | 剪映专业版 | 快速排列分镜、定结构、定节奏 | 上手快，适合 AI 片段快速预览与初筛 |
| 精剪 | DaVinci Resolve | 逐帧修剪、转场、声画对位、最终成片 | 专业时间线，支持 ProRes 中间格式与 ACES 工作流 |

### 2.2 时间线设置

| 参数 | 设定值 | 说明 |
|------|--------|------|
| 帧率 | 24 fps | 电影感标准帧率，与生成帧率一致，避免插帧伪影 |
| 分辨率 | 3840 × 2160（4K UHD） | 母版分辨率，原始 720P 片段按比例放大填满 |
| 宽高比 | 16:9 | 横版母版；竖版为衍生转制（见第六章） |
| 色彩空间 | Rec.709 / ACES | DaVinci 使用 ACES 工作流，输出 Rec.709 |
| 中间格式 | Apple ProRes 422 HQ | 高质量、可多次编码无明显损失 |
| 时间码 | 24 fps（非丢帧） | 全片统一时间码基准 |
| 音频采样率 | 48 kHz | 视频工程标准 |

### 2.3 转场规范

| 转场类型 | 使用场景 | 时长 | 备注 |
|----------|----------|------|------|
| 硬切（Hard Cut） | 默认转场，绝大多数镜头衔接 | 0 | 主力转场，保持节奏干净 |
| 交叉溶解（Cross Dissolve） | 记忆碎片段落、时空转换、梦境过渡 | 0.5s | 仅在分镜表标注"转场"的镜头使用 |
| 黑场（Dip to Black） | 章节结束、片尾 | 0.5–1.0s | 谨慎使用，不超过 2 处 |
| 闪白（Dip to White） | 核心激活、能量爆发瞬间 | 0.2–0.3s | 仅高潮段落 1–2 处 |

> **禁用**：任何花哨转场（如翻页、百叶窗、马赛克、缩放弹跳、光效扫光等）。AI 生成画面本身已具视觉冲击，转场应克制，避免廉价感。

### 2.4 节奏规范

依据《详细分镜表》的 5 个场景，按叙事情绪划分节奏区间：

| 场景 | 叙事段落 | 节奏类型 | 平均单镜时长 | 剪辑策略 |
|------|----------|----------|--------------|----------|
| S01 废墟苏醒 | 开场建立 | 慢节奏 | 3–5s/镜 | 长镜留白，营造孤独与荒凉 |
| S02 飞船残骸 | 探索推进 | 中慢节奏 | 3–4s/镜 | 缓慢积累悬念 |
| S03 核心 chamber | 核心揭示 | 加速 | 2–3s/镜 | 信息密度上升，镜头渐密 |
| S04 记忆碎片 | 情感高潮 | 快切 + 蒙太奇 | 1.5–2.5s/镜 | 蒙太奇叠加，情绪爆发 |
| S05 选择与回响 | 结局升华 | 由快转慢 | 2–4s/镜 | 高潮后回归长镜收束 |

**节奏执行要点**：
- 废墟场景（S01）保留环境音与留白，单镜不少于 3s，让观众"进入"空间。
- 核心场景（S03）起加速，镜头时长从 3s 递减至 2s，配合音乐推进。
- 高潮段落（S04）使用快切，单镜可短至 1.5s，但需保证每镜信息可读。
- 全片避免节奏均匀化，应有明确的"呼吸感"——快慢交替。

### 2.5 剪辑工程规范

- 粗剪导出 H.264 1080P 预览版即可，用于评审节奏与结构。
- 精剪工程必须以 ProRes 422 HQ 中间片工作，导出同格式。
- 所有片段在时间线上**按分镜编号命名轨道/片段**（如 `S01_03`），便于调色与修复阶段定位。
- 精剪定稿后锁定画面（Picture Lock），后续调色、增强、修复不再改动剪辑点；如必须改动，回到精剪工序，版本号递增。

## 三、调色规范

### 3.1 整体风格

**Teal & Orange（青橙色调）电影感**——与《详细分镜表》确立的视觉风格一致。阴影偏青（teal）营造科幻冷峻感，高光偏暖橙（orange）突出人物与光源，形成冷暖对比的立体画面。

### 3.2 DaVinci Resolve 色彩空间与工作流

| 项目 | 设定 |
|------|------|
| 色彩科学 | DaVinci YRGB Color Managed |
| 工作流 | ACES 1.3 |
| ACES 输入色彩空间（IDT） | Rec.709（AI 生成片段按 Rec.709 解释） |
| ACES 工作色彩空间 | ACEScct |
| ACES 输出色彩空间（ODT） | Rec.709 |
| 输出伽马 | Rec.709 Gamma |
| 监看 | Rec.709 校准监视器 / 100% sRGB 显示器 |

> AI 生成片段无真实相机 LOG，按 Rec.709 直接解释即可，无需做 LOG 还原。

### 3.3 一级调色（Primary）

一级调色目标：统一全片曝光、白平衡、对比度，消除不同生成批次间的画面差异。

| 参数 | 调整范围 | 说明 |
|------|----------|------|
| 色温（Color Temp） | −150 至 +150 | 整体偏冷场景向 −100 至 −150，暖光源场景向 +50 至 +100 |
| 色调（Tint） | −50 至 +50 | 微调品绿偏移，AI 片段偶有偏绿，向 +20 至 +40 修正 |
| 曝光（Exposure） | −0.5 至 +0.5 stop | 统一全片曝光基准，暗场补 +0.3，高光压 −0.2 |
| 对比度（Contrast） | 1.05–1.20 | 适度提升，避免 AI 片段发灰 |
| 高光（Highlights） | −20 至 −40 | 压制过曝区域，保留细节 |
| 阴影（Shadows） | +10 至 +30 | 提暗部，但不过度，保留黑场 |
| 白场（Whites） | −10 至 −30 | 防止高光溢出 |
| 黑场（Blacks） | −10 至 −20 | 加深黑场，增加质感 |
| 饱和度（Saturation） | 45–60 | AI 片段常过饱和，整体下调至 50 左右 |

### 3.4 二级调色（Secondary）

二级调色目标：建立 Teal & Orange 风格化色调，同时保护肤色。

| 调整对象 | 限定方式 | 调整方向 | 参数建议 |
|----------|----------|----------|----------|
| 阴影区 | HSL 限定 / 亮度限定 | 偏青（teal） | 色相向 180°–200°，饱和度 +15 至 +25 |
| 高光区 | HSL 限定 / 亮度限定 | 偏暖橙（orange） | 色相向 30°–45°，饱和度 +10 至 +20 |
| 中间调 | 亮度限定 | 保持中性或微暖 | 色相 +5 至 +10 |
| 肤色 | HSL 限定（色相 20°–50°，饱和 20–60） | 保护，反向偏暖 | 色温 +100 至 +200，饱和 −5 至 −10，避免肤色发青 |
| 天空/废墟青光 | HSL 限定 | 强化青色 | 饱和度 +10 至 +20 |
| 光源/能量核心 | HSL 限定（橙黄） | 强化暖橙 | 饱和度 +15 至 +25，亮度 +5 |

**肤色保护要点**：Teal & Orange 易导致肤色发青发灰，必须用 HSL 限定单独圈出肤色区域，反向加暖并适度降饱和，确保艾娃等角色面部自然。

### 3.5 LUT 推荐

| LUT 类型 | 来源 | 用途 |
|----------|------|------|
| 自定义 LUT | 项目自建，导出自 DaVinci | 主用，基于本片调色定稿导出 `.cube`，保证全片一致 |
| Teal & Orange 预设 | 第三方（如 RocketStock、Lutify） | 备用，作为风格起点，需二次微调 |
| ACES Rec.709 ODT | DaVinci 内置 | 技术性 LUT，工作流自动应用 |

> 建议流程：先用 ACES 工作流做一级统一，再用节点树做二级风格化，定稿后导出 `.cube` 作为项目自定义 LUT，存入 `05_Output/Final/SF_TealOrange_vNN.cube`，便于修复与衍生版本复用。

### 3.6 风格化附加层

| 效果 | 强度 | 说明 |
|------|------|------|
| 胶片颗粒（Film Grain） | 15–25 | DaVinci 自带或叠加颗粒素材，增强电影质感，掩盖 AI 平滑感 |
| 暗角（Vignette） | 0.3–0.5 | 轻微暗角，聚焦画面中心，废墟场景可略强 |
| 光晕（Glow/Bloom） | 极弱 | 仅用于能量核心、光源处，强化科幻氛围 |
| 尘埃微粒 | 叠加素材 | 废墟场景叠加尘埃粒子素材，增强空气感 |

## 四、画质增强规范（Topaz Video AI）

AI 生成片段原始分辨率为 720P–1080P，需经 Topaz Video AI 超分至 4K，并做降噪与锐化处理。

### 4.1 模型选择

| 模型 | 适用场景 | 本项目用途 |
|------|----------|------------|
| Proteus v4 | 通用视频超分，适合无主导主体的画面 | 废墟全景、飞船残骸、环境镜头 |
| Iris v2 | 人物为主体的画面，强化面部细节 | 艾娃特写、中近景、对话镜头 |
| Nyx v3 | 降噪专用，处理高噪点/压缩痕迹 | 暗场、低光生成片段、可灵云端压缩片 |
| Theia | 细节增强（备选） | 备用，当 Proteus 效果不佳时尝试 |
| Gaia HQ | 高质量源（HQ）超分 | 不适用（AI 片段非 HQ 源） |

**模型选择决策**：
- 镜头中人物面部占比 ≥ 30% → **Iris**
- 镜头为纯环境/远景/废墟 → **Proteus**
- 片段噪点明显、有压缩块 → 先 **Nyx** 降噪，再超分

### 4.2 输出参数

| 参数 | 设定值 | 说明 |
|------|--------|------|
| 输出分辨率 | 3840 × 2160（4K UHD） | 母版目标分辨率 |
| 缩放倍数 | 2× 或 3×（视原始分辨率） | 720P→4K 约 3×，1080P→4K 约 2× |
| 输出帧率 | 24 fps（不插帧） | 默认保持原帧率，避免插帧伪影 |
| 备选帧率 | 60 fps（如需流畅） | 仅在特定运动镜头需要丝滑感时启用，全片保持 24fps 为主 |
| 插帧模型 | Apollo（如启用 60fps） | 仅运动镜头选用，静态镜头禁用 |
| 输出格式 | Apple ProRes 422 HQ | 中间格式，保留质量供后续工序 |
| 编码器 | Apple ProRes | — |

> **默认不插帧**：24fps 是电影感核心，插帧会破坏 24fps 的运动模糊与质感。仅在评审确认需要"高帧率流畅感"的特定镜头（如核心激活、能量爆发）启用 60fps，且需导演签字确认。

### 4.3 降噪与锐化参数建议

| 参数 | 建议范围 | 说明 |
|------|----------|------|
| 降噪（Reduce Noise） | 5–20 | Proteus/Iris 自带；Nyx 模型可至 30–50；过高会糊化细节 |
| 降噪模式 | Auto / Manual | 暗场手动调高，亮场调低 |
| 锐化（Sharpen） | 10–30 | 适度，过高会产生边缘光晕与"塑料感" |
| 恢复细节（Recover Detail） | 10–25 | 补偿降噪丢失的纹理 |
| 减少模糊（Deblur） | 0–15 | 轻微，仅运动模糊明显时启用 |
| 稳定（Stabilization） | 按需 | 手持感镜头原则上不稳定化，保持真实感；仅明显抖动时启用 |

**参数调优原则**：
- 先降噪后锐化，避免锐化放大噪点。
- 每个模型参数调整后必须预览输出，不可盲批。
- 人物面部镜头优先保细节，宁可保留少量噪点也不要糊化五官。

### 4.4 批处理工作流

由于全片 24 个镜头需逐镜处理，采用批处理提高效率：

1. **按场景分组**：将镜头按 S01–S05 分组，每组使用统一的模型与参数预设。
2. **建立预设**：在 Topaz 中保存预设：
   - `SF_Proteus_Env_v01`（环境镜头）
   - `SF_Iris_Face_v01`（人物镜头）
   - `SF_Nyx_Denoise_v01`（降噪预处理）
3. **批处理队列**：将同组镜头加入队列，统一应用预设输出。
4. **抽检**：每组批处理完成后，随机抽取 2–3 个镜头全片预览，确认无偏色、无伪影。
5. **命名输出**：按命名规范输出至 `05_Output/Final/`，每镜独立文件。

| 步骤 | 操作 | 输出位置 |
|------|------|----------|
| 1 | 按场景分组导入 | — |
| 2 | 应用对应预设 | — |
| 3 | 批处理渲染 | `05_Output/Final/SXX_GYY/SF_SXX_GYY_upscaled_vNN.mov` |
| 4 | 抽检全片预览 | — |
| 5 | 不合格镜头单独重调参数 | 版本号递增 |

> **注意**：Topaz 输出偶有偏色（见 SOP 常见问题排查），抽检时重点核对肤色与中性灰，发现偏色立即更换模型或回退参数。

## 五、瑕疵修复规范（After Effects）

AI 生成视频的典型瑕疵需在 After Effects 中逐帧或逐段修复。本阶段在 4K 增强画面上进行。

### 5.1 常见问题与修复方法

| 问题 | 表现 | 修复方法 | 工具/技术 |
|------|------|----------|-----------|
| 闪烁（Flicker） | 帧间亮度/色彩跳变 | 时域平滑、逐帧均化 | AE 时间差值、Neat Video、Flicker Free 插件 |
| 穿帮（Continuity Error） | 服装/道具/背景帧间不一致 | 遮罩替换、参考帧修补 | 遮罩（Mask）、跟踪（Track）、合成 |
| 手指变形（Hand Mutation） | 多指、融合、扭曲 | 逐帧修补、替换手部 | 逐帧绘制（Roto Brush）、图章工具、参考帧 |
| 角色不一致（Identity Drift） | 面部特征帧间变化 | 关键帧替换、面部锚定 | 面部跟踪、替换参考帧、动态变形 |
| 物体融化/变形 | 物体边缘流动、结构崩坏 | 遮罩替换、稳定 | 遮罩 + 跟踪 + 替换素材 |
| 水印/伪影 | 边缘伪影、压缩块 | 局部修补 | 图章、内容识别填充 |

### 5.2 修复技术细则

#### 5.2.1 逐帧修补（Rotoscoping）
- 使用 Roto Brush 2.0 圈选问题区域，逐帧检查边缘。
- 适用于手指变形、物体边缘崩坏等局部问题。
- 修补素材优先取自同镜头的"好帧"，做时域对齐与变形匹配。

#### 5.2.2 遮罩替换（Mask Replacement）
- 对持续穿帮的区域（如背景闪烁的物体）做遮罩，替换为稳定素材。
- 配合 3D 摄像机跟踪（3D Camera Tracker）确保遮罩随镜头运动。

#### 5.2.3 动态跟踪（Motion Tracking）
- 面部不一致问题：跟踪面部特征点，将"标准脸"参考帧变形贴合到问题帧。
- 使用 AE 内置跟踪器或 Mocha AE 进行平面跟踪。

#### 5.2.4 闪烁处理
- 全局闪烁：使用 Flicker Free 或 Neat Video 时域降噪。
- 局部闪烁：遮罩限定区域后做时域平滑。

### 5.3 修复优先级

| 优先级 | 问题类型 | 处理原则 |
|--------|----------|----------|
| P0 | 角色面部严重不一致、手指明显畸形 | 必须修复，影响观众认同 |
| P1 | 持续闪烁、明显穿帮 | 应修复，影响沉浸感 |
| P2 | 边缘伪影、轻微变形 | 视时长与成本决定，可保留 |
| P3 | 一闪而过的瑕疵（< 3 帧） | 可忽略，观众不易察觉 |

### 5.4 何时放弃修复改为重新生成

修复耗时与生成耗时的权衡是关键决策。出现以下情况应**放弃修复，回到视频生成工序重新生成该镜头**：

| 判定条件 | 处理 |
|----------|------|
| 单镜修复工时 > 重新生成工时 × 1.5 | 重新生成 |
| 角色面部全镜段不一致（非局部） | 重新生成，加强 IPAdapter 锚点 |
| 手指/肢体变形贯穿整镜 | 重新生成，切分镜头或换可灵首尾帧约束 |
| 闪烁无法用时域平滑消除 | 重新生成，降低 Wan2.2 Denoise |
| 修复后画面"塑料感"明显 | 重新生成，修复优先级低于画面自然度 |

> **决策记录**：每次"放弃修复改为重新生成"的决策需记录在 `06_Research/failure_cases.md`，注明镜头号、问题、原因、重生成参数，供后续迭代参考。

### 5.5 修复工程规范

- 每个需修复的镜头建立独立 AE 工程，命名 `SF_SXX_GYY_repair_vNN.aep`。
- 修复输出为 ProRes 422 HQ，存入 `05_Output/Final/`。
- 修复完成后与增强版做 A/B 对比，确认无新引入瑕疵。

## 六、输出与交付规范

### 6.1 母版格式

| 参数 | 设定值 |
|------|--------|
| 格式 | Apple ProRes 422 HQ |
| 容器 | MOV |
| 分辨率 | 3840 × 2160（4K UHD） |
| 帧率 | 24 fps |
| 宽高比 | 16:9 |
| 色彩空间 | Rec.709 |
| 音频 | 48 kHz，立体声，PCM 24bit |
| 整体响度 | −14 LUFS（见 6.4） |
| 文件名 | `SF_final_4k_ProRes_vNN.mov` |

### 6.2 发布格式（横版主版）

| 参数 | 设定值 |
|------|--------|
| 格式 | H.265 / HEVC |
| 容器 | MP4 |
| 分辨率 | 3840 × 2160（4K UHD） |
| 帧率 | 24 fps |
| 码率控制 | CRF 18–20 |
| 预设 | Slow（编码质量优先） |
| 像素格式 | yuv420p |
| 音频 | AAC 320 kbps，48 kHz，立体声 |
| 文件名 | `SF_final_4k_vNN.mp4` |

> CRF 18 接近视觉无损，CRF 20 兼顾质量与体积。成片建议 CRF 18，预览版可用 CRF 20。

### 6.3 竖版衍生格式

用于短视频平台（抖音、视频号、小红书）发布。

| 参数 | 设定值 |
|------|--------|
| 格式 | H.264 / AVC |
| 容器 | MP4 |
| 分辨率 | 1080 × 1920（竖版 9:16） |
| 帧率 | 24 fps |
| 码率控制 | CRF 20–22 |
| 转制方式 | 横版母版裁剪/缩放 + 上下模糊填充或重新构图 |
| 音频 | AAC 256 kbps，48 kHz，立体声 |
| 文件名 | `SF_final_vertical_1080x1920_vNN.mp4` |

**竖版转制要点**：
- 不可简单等比缩放导致主体过小，需逐镜重新构图（Pan & Scan）。
- 人物镜头以面部为中心裁切；全景镜头可上下加模糊填充保留信息。
- 字幕/标题需重新排版至竖版安全区。

### 6.4 音频规范

| 参数 | 设定值 |
|------|--------|
| 采样率 | 48 kHz |
| 声道 | 立体声（Stereo） |
| 位深 | 24 bit（母版）/ 16 bit（发布） |
| 整体响度 | −14 LUFS |
| 响度容差 | ±1 LUFS |
| True Peak | −1 dBTP |
| 噪声底 | ≤ −60 dBFS |

**混音平衡建议**：

| 音轨 | 响度参考 | 说明 |
|------|----------|------|
| 对白（配音） | −12 至 −10 LUFS | 最高优先级，确保清晰可懂 |
| 配乐 | −20 至 −16 LUFS | 衬托对白，高潮段落可升至 −14 LUFS |
| 音效 | −24 至 −18 LUFS | 环境音与动作音效，不抢对白 |
| 整体混音 | −14 LUFS | 平台发布标准（YouTube/抖音推荐） |

### 6.5 字幕与元数据

| 文件 | 格式 | 说明 |
|------|------|------|

## 八、质量检查清单

每道工序交付前，由责任人逐项核对；成片交付前由 PM 组织全片复核。

### 8.1 剪辑检查清单

- [ ] 时间线帧率为 24fps，分辨率为 3840×2160
- [ ] 全片时长符合 3–5 分钟目标（参考 4 分 10 秒）
- [ ] 所有镜头按分镜表排列，无遗漏、无多余
- [ ] 转场仅使用硬切 / 0.5s 交叉溶解 / 黑场 / 闪白，无花哨转场
- [ ] 节奏符合场景规范（废墟慢、核心加速、高潮快切）
- [ ] 声画基本对位（精剪阶段）
- [ ] 无黑场、无断帧、无时间线空隙
- [ ] 片头片尾完整（标题卡、演职员）

### 8.2 调色检查清单

- [ ] ACES 工作流正确配置（IDT/ODT 设置无误）
- [ ] 全片曝光、白平衡统一，无批次差异
- [ ] Teal & Orange 风格一致（阴影青、高光橙）
- [ ] 肤色保护到位，无发青发灰
- [ ] 高光未溢出（波形图顶部无切割）
- [ ] 黑场未塌陷（波形图底部保留细节）
- [ ] 饱和度适中，无过饱和
- [ ] 胶片颗粒、暗角等风格化层强度适中
- [ ] 已导出项目自定义 LUT（`.cube`）

### 8.3 画质增强检查清单

- [ ] 输出分辨率为 3840×2160
- [ ] 帧率为 24fps（除非明确启用 60fps）
- [ ] 模型选择正确（人物用 Iris，环境用 Proteus，降噪用 Nyx）
- [ ] 无偏色（重点核对肤色与中性灰）
- [ ] 无插帧伪影、无边缘光晕
- [ ] 降噪未糊化细节（面部五官清晰）
- [ ] 锐化未产生"塑料感"
- [ ] 抽检通过（每组随机 2–3 镜全片预览）

### 8.4 瑕疵修复检查清单

- [ ] P0 级问题（面部不一致、手指畸形）已全部修复
- [ ] P1 级问题（闪烁、穿帮）已修复
- [ ] 修复未引入新瑕疵（A/B 对比确认）
- [ ] 修复后画面自然，无"塑料感"
- [ ] 放弃修复的镜头已记录并重新生成
- [ ] 修复工程文件已归档

### 8.5 音频检查清单

- [ ] 采样率 48 kHz，立体声
- [ ] 整体响度 −14 LUFS（±1 LUFS）
- [ ] True Peak ≤ −1 dBTP
- [ ] 对白清晰可懂，不被配乐/音效掩盖
- [ ] 配乐情绪与画面匹配，高潮段落同步
- [ ] 音效定位合理，无突兀
- [ ] 无爆音、无底噪异常
- [ ] 声画同步，无明显错位

### 8.6 输出与交付检查清单

- [ ] 母版为 ProRes 422 HQ，4K，24fps
- [ ] 发布版为 H.265，4K，CRF 18–20
- [ ] 竖版衍生为 H.264，1080×1920
- [ ] 文件命名符合规范（`SF_final_*`）
- [ ] 中英文字幕已生成并校对
- [ ] 封面图（横竖）已制作
- [ ] 元数据已写入
- [ ] 母版已归档至 `05_Output/Final/` 与网盘
- [ ] Git tag 已打（`release/v04-final-4k`）
- [ ] 校验清单（MD5/SHA256）已生成

### 8.7 全片终审清单（PM + 导演）

- [ ] 叙事完整，情绪连贯
- [ ] 视觉风格统一（Teal & Orange 全片一致）
- [ ] 角色一致性可接受（盲测通过）
- [ ] 无明显 AI 痕迹（手指、变形、闪烁）
- [ ] 节奏张弛有度，无拖沓或仓促
- [ ] 音画体验完整，沉浸感达标
- [ ] 符合 3–5 分钟时长要求
- [ ] 评审签字归档
