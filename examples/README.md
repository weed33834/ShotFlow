# Examples

> Ready-to-reference samples for understanding how ShotFlow is used.

---

## Contents

| File / Dir | Purpose |
|------|------|
| `echo-of-singularity/` | Full case study: proposal, production log, character bible, shot tracker |
| `character_prompts.md` | Ava character-consistency prompt example |
| `storyboard_sample.md` | Simplified storyboard for the first 3 shots |
| `comfyui_api_payload.json` | Minimal ComfyUI API payload example |
| `env.example` | Minimum environment variable template |

---

## Quick reference

1. Start with `echo-of-singularity/README.md` to see how a full case is planned and tracked end-to-end.
2. Then read `character_prompts.md` to see how character anchors are written.
3. Then `storyboard_sample.md` for the storyboard format.
4. Copy `env.example` to `.env` and fill in your API keys.
5. Adapt `comfyui_api_payload.json` for your own ComfyUI workflow API calls.
