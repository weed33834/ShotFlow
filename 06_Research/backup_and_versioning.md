## 一、资产分类与存储策略

按资产类型、单文件体积、访问频率与重要性进行分级。分级直接决定存储介质与备份策略。

### 1.1 分级定义

| 等级 | 名称 | 说明 | 典型存储介质 |
|------|------|------|--------------|
|  | 热数据 | 当前正在生产/剪辑，需高频随机读写 | 本地 NVMe SSD |
| 温 | 温数据 | 阶段性产出，需随时调用但不再频繁修改 | 本地 NAS / SATA SSD |
| 冷 | 冷数据 | 已定稿归档，极少访问，仅作留档与回溯 | 外置 HDD / 云盘冷存储 |

### 1.2 资产分类表

| 资产类型 | 示例文件 | 单文件体积 | 数量级 | 总体积估算 | 重要性 | 分级 | 主存储位置 |
|----------|----------|-----------|--------|-----------|--------|------|-----------|
| ComfyUI 工作流 JSON | `Flux_Character_Consistency.json`、`Wan22_Dual_Expert_Video.json` | < 1 MB | 10–20 | < 20 MB | 极高 | 热数据 | 本地 SSD + Git LFS |
| 脚本与文档 | 剧本、SOP、角色圣经、分镜表 | < 1 MB | 30–50 | < 50 MB | 极高 | 热数据 | 本地 SSD + Git |
| 角色参考图 | Flux 出图、IPAdapter 锚点图 | 1–5 MB | 50–100 | ~300 MB | 高 | 热数据 | 本地 SSD + Git LFS |
| 场景关键帧 | 24+ 张关键帧 PNG | 2–8 MB | 20–40 | ~200 MB | 高 | 热数据 | 本地 SSD + Git LFS |
| 原始视频片段 | Wan2.2 / 可灵生成原始 MP4 | 20–200 MB | 40–60 | ~8 GB | 高 | 温数据 | NAS + 网盘 |
| 粗剪工程 | 剪映工程文件 + 代理素材 | 5–50 MB（工程） | 3–5 版 | ~2 GB | 中高 | 温数据 | NAS |
| 精剪工程 | DaVinci 工程 + 调色 LUT | 10–100 MB | 3–5 版 | ~1 GB | 高 | 温数据 | NAS + 网盘 |
| 配音音频 | ElevenLabs 输出 WAV/MP3 | 1–10 MB | 30–60 | ~300 MB | 高 | 温数据 | NAS + Git LFS（小文件） |
| 配乐 | Suno 输出 + BGM | 3–20 MB | 5–15 | ~200 MB | 中高 | 温数据 | NAS + 网盘 |
| 音效素材库 | 通用 SFX | 0.5–5 MB | 100+ | ~500 MB | 中 | 温数据 | NAS |
| 4K 最终成片 | 超分后 MP4 / ProRes | 1–10 GB | 1–3 版 | ~15 GB | 极高 | 冷数据 | 外置 HDD + 云盘 |
| 中间产物 | 调色中间片、AE 合成缓存 | 100 MB–2 GB | 多次 | ~30 GB | 低 | 温数据 | NAS（可清理） |
| 模型权重 | Flux / Wan2.2 / IPAdapter 模型 | 1–30 GB | 5–10 | ~120 GB | 中（可重下） | 冷数据 | NAS（不进 Git） |
| 字幕 / 元数据 | SRT、章节标记 | < 1 MB | 5–10 | < 5 MB | 中 | 热数据 | Git |

**合计估算**：热数据 < 1 GB；温数据 ~45 GB；冷数据 ~135 GB；总计约 **180–250 GB**（含冗余与中间产物可达 300 GB+）。

## 二、三层备份策略（3-2-1 原则）

遵循 **3-2-1 原则**：**3** 份副本、**2** 种不同介质、**1** 份异地。

### 2.1 第一层：本地 NAS / 外置硬盘（主存储）

| 项目 | 配置建议 |
|------|----------|
| 设备 | 4 盘位 NAS（如群晖 DS923+ / 极空间 Z4Pro）或 2×8TB 外置 HDD |
| RAID | RAID 1（镜像）或 RAID 5（兼顾容量与冗余） |
| 可用容量 | ≥ 4 TB（预留 2 倍项目体积用于版本叠加） |
| 用途 | 温数据主存储、热数据本地镜像、冷数据近线归档 |
| 同步方式 | 工作机 SSD 每日定时 rsync / Synology Drive 同步至 NAS |

### 2.2 第二层：云盘备份

针对不同资产选择不同云盘，平衡速度、容量与成本。

| 云盘 | 适用资产 | 说明 |
|------|----------|------|
| 百度网盘 SVIP | 原始视频、4K 成片、模型权重 | 容量大（5 TB+），下载限速需会员 |
| 阿里云盘 | 精剪工程、配音、配乐 | 上传下载不限速，适合中等体积 |
| Google Drive | 工作流 JSON、脚本、文档、小型参考图 | 与 Git 仓库协同，海外协作友好 |

> **注意**：云盘不保证永久可用，重要资产务必同时保留本地与异地副本。模型权重若可从 HuggingFace 重新下载，可仅保留下载链接清单，节省云盘空间。

### 2.3 第三层：异地冷备份（可选但推荐）

| 项目 | 配置建议 |
|------|----------|
| 介质 | 1× 4TB 便携 HDD（如西部数据 My Passport） |
| 频率 | 每个里程碑节点（角色定稿、关键帧定稿、粗剪定稿、成片交付）一次 |
| 存放 | 异地（家中另一地点 / 团队成员处 / 银行保管箱） |
| 校验 | 每次备份后生成 MD5/SHA256 校验清单，归档时核对 |

## 三、版本控制方案

按文件体积与类型分流：**小文件进 Git，大文件进网盘 + 命名版本号**。

### 3.1 Git LFS 管理范围（≤ 50 MB 单文件）

| 资产 | 管理方式 | 说明 |
|------|----------|------|
| ComfyUI 工作流 JSON | Git 普通文本 | 可 diff，便于追踪节点改动 |
| 剧本 / SOP / 角色圣经 | Git 普通文本 | Markdown，支持协作 review |
| 分镜表 / 参数调优记录 | Git 普通文本 | 表格类 Markdown |
| 字幕 SRT | Git 普通文本 | 可 diff |
| 角色参考图 / 关键帧 PNG | Git LFS | 二进制，LFS 避免仓库膨胀 |
| 配音音频（小文件） | Git LFS | WAV/MP3 ≤ 50 MB |

### 3.2 网盘 + 命名版本号管理范围（> 50 MB 单文件）

| 资产 | 管理方式 | 说明 |
|------|----------|------|
| 原始视频片段 | 网盘 + 命名版本号 | 不进 Git，靠文件名区分版本 |
| 粗剪 / 精剪工程 | 网盘 + 命名版本号 | 工程目录整体打包上传 |
| 4K 最终成片 | 网盘 + 命名版本号 | 每版独立保留 |
| 模型权重 | 网盘 / HuggingFace 链接 | 不进 Git，记录来源 URL 与版本 |
| 中间产物 | NAS（不备份） | 可再生，定期清理 |

### 3.3 Git 分支策略

| 分支 | 用途 |
|------|------|
| `main` | 稳定版本，仅合并已验证的工作流与文档 |
| `dev` | 日常开发，工作流迭代、剧本修订 |
| `feature/<内容>` | 如 `feature/char-linmo`、`feature/scene-03` |
| `release/vXX` | 里程碑节点快照，对应网盘大文件版本号 |

## 四、命名与版本号规范

### 4.1 文件命名规范

**通用格式**：

```
SF_场景号_镜头号_版本_描述.扩展名
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `PS` | 项目固定前缀（ShotFlow） | `PS` |
| 场景号 | 两位数字，`S00` 表示通用资产 | `S03` |
| 镜头号 | 两位数字，`G00` 表示场景级资产 | `G05` |
| 版本 | `v01`、`v02`...`final` | `v03` |
| 描述 | 简短英文/拼音，下划线分隔 | `linmo_closeup` |

**示例**：

```
SF_S03_G05_v02_linmo_closeup.png        # 第3场第5镜第2版 林墨特写关键帧
SF_S03_G05_final_linmo_closeup.mp4      # 第3场第5镜最终版 视频
SF_S00_G00_v01_workflow_wan22.json      # 通用 Wan2.2 工作流第1版
SF_S00_G00_v02_dubbing_linmo.wav        # 林墨配音第2版
```

### 4.2 版本号规则

| 版本标记 | 含义 | 说明 |
|----------|------|------|
| `v01`、`v02`、`v03`... | 迭代版本 | 每次重大修改递增，两位数字 |
| `final` | 最终定稿 | 经评审通过，不再修改 |
| `final_4k` | 4K 成片定稿 | 超分后最终版 |
| `draft` | 草稿 | 内部初稿，未评审 |

> 禁止使用 `最新版`、`修改版`、`最终版2` 等模糊命名。每次保存新版本时**保留旧版本**，不覆盖。

### 4.3 日期目录规范

用于按日期归档的目录（如每日产出、网盘上传包）：

```
YYYY-MM-DD_内容描述
```

**示例**：

```
2026-06-23_角色参考图_林墨
2026-06-25_场景03_视频生成批次1
2026-06-28_粗剪_v01
2026-07-01_精剪_v02_调色
2026-07-05_成片_final_4k
```

## 七、灾难恢复流程

### 7.1 故障分级与响应

| 级别 | 故障描述 | RTO（恢复时间目标） | RPO（数据丢失容忍） |
|------|----------|--------------------|--------------------|
| P0 | 本地 SSD 全损（工作机故障） | < 4 小时 | < 1 天 |
| P1 | NAS 故障（RAID 降级或损坏） | < 24 小时 | < 1 天 |
| P2 | 云盘账号异常 / 文件丢失 | < 48 小时 | 0（多副本） |
| P3 | 异地 HDD 损坏 | < 7 天 | 0（多副本） |

### 7.2 恢复流程

#### 场景 A：工作机 SSD 故障（P0）

1. 从 NAS 恢复最近一次同步的热数据（工作流、文档、参考图）。
2. `git clone` 远程仓库，`git lfs pull` 拉取 LFS 资产。
3. 从 NAS / 云盘恢复当前正在处理的视频与工程。
4. 核对 `99_Archive/Checksums/` 校验清单，确认数据完整。
5. 在新工作机重建 ComfyUI 环境（参考 `04_SOP/sop_shotflow.md`）。

#### 场景 B：NAS 故障（P1）

1. 评估 RAID 降级情况，尝试在线重建。
2. 若 RAID 不可恢复，从云盘 + 异地 HDD 重建 NAS。
3. 优先恢复当前生产所需的温数据（视频、工程）。
4. 冷数据（成片、模型）按需恢复，可后置。
5. 故障期间，工作机 SSD 暂作主存储，每日手动上传云盘。

#### 场景 C：误删 / 误覆盖（任意级别）

1. **立即停止**对相关目录的写入。
2. Git 管理资产：`git reflog` + `git checkout` 恢复历史版本。
3. NAS 资产：使用 NAS 回收站或快照（如 Synology Snapshot Replication）。
4. 云盘资产：从云盘历史版本 / 回收站恢复。
5. 记录事故原因，更新 SOP 防止复发。

### 7.3 恢复演练

- **频率**：每个里程碑节点前进行一次桌面演练，每季度进行一次实际恢复演练。
- **内容**：随机抽取 5 个资产，模拟从零恢复，验证备份可用性与流程时效。
- **记录**：演练结果写入 `06_Research/` 下的演练报告。

## 八、存储空间规划与预算

### 8.1 存储空间规划

| 存储层 | 介质 | 容量规划 | 用途 | 备注 |
|--------|------|----------|------|------|
| 工作机本地 | NVMe SSD | 2 TB | 热数据 + 当前处理中的温数据 | 高速读写 |
| 本地 NAS | HDD ×4 (RAID 5) | 4 × 4 TB = 12 TB（可用 8 TB） | 温数据主存储 + 冷数据近线 | 预留版本叠加空间 |
| 外置 HDD（异地） | 便携 HDD | 4 TB | 冷备份快照 | 每里程碑更新 |
| 云盘 | 百度网盘 SVIP | 5 TB+ | 大文件冷备份 | 年费会员 |
| 云盘 | 阿里云盘 | 2 TB+ | 中等文件备份 | 免费或会员 |
| 云盘 | Google Drive | 100 GB+ | Git 协作 + 小文件 | Google One 基础版 |

### 8.2 预算估算（按 GB / 按年）

| 项目 | 规格 | 单价 | 数量/周期 | 小计（CNY） |
|------|------|------|----------|------------|
| NVMe SSD | 2 TB | ¥0.5/GB | 一次性 | ~¥1,000 |
| NAS 主机 | 4 盘位 | — | 一次性 | ~¥4,000 |
| NAS 硬盘 ×4 | 4 TB 企业级 | ¥0.15/GB | 一次性 | ~¥2,400 |
| 异地便携 HDD | 4 TB | ¥0.18/GB | 一次性 | ~¥720 |

### 8.3 空间节省策略

### 9.3 `.gitattributes` 配置示例

```gitattributes
# Git LFS 追踪规则
*.png filter=lfs diff=lfs merge=lfs -text
*.jpg filter=lfs diff=lfs merge=lfs -text
*.jpeg filter=lfs diff=lfs merge=lfs -text
*.wav filter=lfs diff=lfs merge=lfs -text
*.mp3 filter=lfs diff=lfs merge=lfs -text
*.psd filter=lfs diff=lfs merge=lfs -text
*.clip filter=lfs diff=lfs merge=lfs -text
*.drp filter=lfs diff=lfs merge=lfs -text

# 文本文件强制 LF 换行
*.md text eol=lf
*.json text eol=lf
*.py text eol=lf
*.sh text eol=lf
*.srt text eol=lf
*.txt text eol=lf
```

### 9.4 里程碑打 Tag 示例

```bash
# 角色定稿
git tag -a release/v01-char-final -m "角色资产定稿"

# 关键帧定稿
git tag -a release/v02-keyframe-final -m "场景关键帧定稿"
```
