# 09_Release — Release and showcase

Templates and specs for the final release phase: distribution package, poster
specs, presentation, tutorial, and release checklist. Templates are blank —
fill them in per project. The example film's specs are filled in.

## Files

### Templates (blank)

- [`release_checklist.md`](./release_checklist.md) — pre-release checklist covering
  media, docs, workflows, and platform-specific requirements.
- [`tutorial_template.md`](./tutorial_template.md) — tutorial / write-up template for
  publishing the workflow to the community.
- [`presentation_template.md`](./presentation_template.md) — slide deck template for
  project showcase / defense.

### Example film specs (filled in)

- [`distribution_kit.md`](./distribution_kit.md) — per-platform distribution
  package spec (Bilibili / YouTube / 抖音 / 小红书 / 视频号 / Instagram /
  film festival): master files, cover, title/description templates, tags, AIGC
  disclosure rules, license declaration.
- [`poster_spec.md`](./poster_spec.md) — per-platform cover & poster visual spec:
  sizes, color/font rules, composition principles, Flux.1 prompt templates,
  generation → upscale → grade → typesetting workflow.

## Usage

The example film specs are filled in for *Echo of the Singularity* and serve as
both the actual release plan and a worked example for new projects. The blank
templates are filled in during phase 4 (week 6) of the project schedule. The
release checklist gates the final delivery — see
[`04_SOP/sop_shotflow.md`](../04_SOP/sop_shotflow.md) for the full release procedure.
