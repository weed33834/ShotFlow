01

What this solves

The walls every AIGC short film hits, every single time.

Character drift

The same character looks like a different person in every shot. Reference images drift, anchors slip, faces re-roll.

Motion artifacts

Footage flickers, warps, and moves in ways nothing should move. Hands fuse, edges breathe, physics walks off the set.

Prompt drift

The prompt and the storyboard drift apart, and the gap only shows up in the edit bay — when half the shots are already rendered.

Lost parameters

Parameters vanish between sessions. The one good take can never be rerun, because nobody remembers what produced it.

File chaos

Files get renamed, overwritten, lost. A small team burns more time hunting assets than making them.

ShotFlow ties it together

One pipeline so a result is reproducible and a team can actually collaborate. Template, not product — swap in a story and go.

03

Quick start

Two ways in. Docker to look, source to work.

Fastest look. Backend, worker, Postgres, Redis in one command.

Terminal 
# clone
git clone https://github.com/MS33834/ShotFlow.git
cd ShotFlow

# up — Postgres + Redis + backend + Celery worker
docker compose up -d

# API docs (Swagger)
open http://localhost:8000/docs

ComfyUI and model weights aren't bundled (license + size). Pull them with bash 08_Automation/deploy_comfyui.sh on a GPU host.

For actually making a film. Needs an NVIDIA GPU (RTX 4090 24GB recommended).

Terminal 
# clone
git clone https://github.com/MS33834/ShotFlow.git
cd ShotFlow
cp .env.example .env     # fill API keys

# ComfyUI + nodes
bash 08_Automation/deploy_comfyui.sh

# deps + structure check
make setup
make check

# preview the pipeline without rendering
python 08_Automation/preflight_check.py --dry-run
python 08_Automation/batch_keyframe_gen.py --dry-run
python 08_Automation/storyboard_to_video.py --dry-run

Every generation script takes --dry-run. Run it first — it prints the full plan without calling ComfyUI or cloud APIs.

New here? The step-by-step tutorial walks from an empty repo to a 4K master, one command at a time.

04

Web platform

For team members who don't want a terminal.

Backend API

FastAPI

FastAPI + SQLAlchemy + Celery. SSE push for real-time queue status.

  • POST /api/v1/queue submit render
  • GET /api/v1/queue/stream/events SSE live status
  • /api/v1/shots /keyframes /workflows
  • /api/v1/health DB + Redis

Interactive docs at /docs (Swagger) and /redoc.

Frontend Console

React

React 18 + Vite + Ant Design Pro. Eleven routes, end-to-end typed.

  • /dashboard health + queue + projects
  • /queue SSE-driven render queue
  • /workflow-configs YAML params + provider scoring
  • /assets /audio /qa /case-studies

cd frontend && npm run dev — proxies to :8000.

Simulate mode is on by default — every service returns mock output, so the whole chain runs without a GPU. Flip SIMULATE_MODE=false on a GPU host to hit real backends.

The film that proves it

Echo of the Singularity — not a demo reel, a finished short.

CASE STUDY

奇点回响

ECHO OF THE SINGULARITY

3–5 min 4K AIGC

A complete AIGC short film lives in the repo as a worked example. Script, character bible, 24-shot storyboard, 29 keyframes, 24 video clips, voice bibles, cue sheet, EDL, color notes, mix notes, asset manifest, subtitles, credits, delivery specs — every artifact a reviewer would expect from a finished short.

The rendered video files themselves are not committed (large, several use NC-licensed model outputs). The paperwork is the reference — run the pipeline end-to-end to populate the actual media, then follow assembly_guide.md to lock the master.
