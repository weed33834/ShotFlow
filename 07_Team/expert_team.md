# ShotFlow — 专家团队名册

> 本项目的执行依赖一支跨学科专家团队。以下角色按职责划分，可根据实际成员能力合并或拆分。

## 团队架构图

```
项目导师/教师（外部督导）
        │
        ▼
   项目制片人/PM
   ┌────────┬────────┬────────┐
   ▼        ▼        ▼        ▼
技术总监  创意总监  后期总监  质量总监
   │        │        │        │
   ▼        ▼        ▼        ▼
AI算法    导演/编剧  剪辑调色  QA/测试
工程师    美术指导   声音设计  运维/部署
```

## 1. 项目导师 / 教师（外部督导）

| 项目 | 说明 |
|------|------|
| 职责 | 全程跟踪项目进展，审阅里程碑交付物，提供学术/专业指导，把控项目方向 |
| 参与方式 | 每周 1 次项目例会、关键节点评审、最终成片审看 |
| 交付物签字 | 阶段验收表、最终项目报告 |
| 所需经验 | 熟悉 AIGC、影视制作或项目管理教学/产业经验 |

## 2. 项目制片人 / PM

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 经验要求 | 3 年以上项目管理经验，熟悉敏捷/瀑布混合管理 |
| 核心职责 | 制定排期、协调资源、跟踪进度、风险管理、对外沟通、预算控制 |
| 关键交付 | 项目计划书、周报、风险清单、预算执行表 |
| 工具 | Notion / 飞书项目 / Excel |

## 3. 技术总监（资深开发者）

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 经验要求 | 5 年以上 Python/CUDA/深度学习部署经验，熟悉 ComfyUI、Diffusers、TensorRT |
| 核心职责 | 本地 ComfyUI 环境搭建、模型选型与量化、工作流性能优化、云端混合架构设计 |
| 关键交付 | 技术架构文档、ComfyUI 工作流 JSON、部署脚本、性能测试报告 |
| 技术栈 | Python 3.12、CUDA 12.x、PyTorch、ComfyUI、TensorRT、Docker（可选） |

## 4. AI 算法工程师

| 项目 | 说明 |
|------|------|
| 人数 | 1–2 人 |
| 经验要求 | 2 年以上生成式 AI 经验，熟悉 Flux/SD/Wan 系列模型、LoRA/IPAdapter/PuLID |
| 核心职责 | 角色一致性方案调优、视频生成参数调优、失败案例分析与修复 |
| 关键交付 | 模型评测报告、参数调优记录、失败案例库 |

## 5. AI 美术指导

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 经验要求 | 3 年以上概念设计/原画经验，精通 Midjourney/Flux/Stable Diffusion 提示词工程 |
| 核心职责 | 角色圣经制定、关键帧设计、风格统一、AI 出图质量把控 |
| 关键交付 | 角色三视图、20+ 关键帧、风格指南、盲测报告 |

## 6. 导演 / 编剧

| 项目 | 说明 |
|------|------|
| 人数 | 1–2 人 |
| 经验要求 | 有短片/微电影编剧或导演经验，理解科幻叙事与视听语言 |
| 核心职责 | 世界观构建、剧本撰写、分镜脚本、剪辑叙事节奏把控 |
| 关键交付 | 剧本、分镜表、导演阐述、粗剪/精剪审看意见 |

## 7. 后期总监

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 经验要求 | 3 年以上视频后期经验，熟悉达芬奇/剪映/After Effects/Topaz |
| 核心职责 | 粗剪、调色、画质增强、特效合成、最终母版输出 |
| 关键交付 | 粗剪版本、调色 LUT、4K 母版、后期修复记录 |

## 8. 声音设计师 / 作曲

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 经验要求 | 有影视配乐、音效设计或 AI 音频工具（Suno/Udio/ElevenLabs）使用经验 |
| 核心职责 | 配音、配乐、环境音效、混音母版 |
| 关键交付 | 对白音轨、背景音乐、音效库、最终混音 |

## 9. 质量总监 / QA

| 项目 | 说明 |
|------|------|
| 人数 | 1 人 |
| 核心职责 | 制定质量标准、组织盲测、审核角色一致性、检查物理逻辑错误、输出 QA 报告 |
| 关键交付 | QA 检查表、盲测报告、Bug 清单、验收签字 |

## 10. 运维 / 部署工程师

| 项目 | 说明 |
|------|------|
| 人数 | 1 人（可由技术总监兼任） |
| 核心职责 | 云端 GPU 弹性扩容、模型/素材备份、发布平台上传、工作流打包分发 |
| 关键交付 | 云端部署脚本、资产备份方案、发布链接 |

## 团队会议节奏

| 会议 | 频率 | 参与者 | 目的 |
|------|------|--------|------|
| 师生同步会 | 每周 1 次 | 全体成员 + 教师 | 汇报进度、确认里程碑、获取反馈 |
| 技术站会 | 每日 15 分钟 | 技术/算法/运维 | 阻塞同步、性能问题 |
| 创意评审会 | 每周 2 次 | 导演/美术/后期/声音 | 关键帧、镜头、粗剪审看 |
| QA 验收会 | 每阶段 1 次 | QA + 教师 + PM | 阶段交付物签字 |
