# 项目协作工具建立指南

> 本指南用于 AIGC 视频项目启动阶段建立团队沟通与协作环境。  
> 以 ShotFlow /《奇点回响》为示例，实际项目请按团队规模与预算调整。

---

## 一、推荐工具组合

| 用途 | 推荐工具 | 备选 | 说明 |
|------|----------|------|------|
| 即时通讯 | 飞书 / 钉钉 / 企业微信 | Slack / Discord | 日常沟通、文件快速同步 |
| 任务看板 | 飞书项目 / Notion / Trello | Jira / Linear | 任务分派、进度跟踪 |
| 文档协作 | 飞书文档 / Notion / 语雀 | Google Docs / Confluence | 计划书、SOP、会议纪要 |
| 文件存储 | 飞书云盘 / 阿里云盘 / NAS | Dropbox / Google Drive | 大文件、资产备份 |
| 代码/工作流托管 | GitHub + GitCode | GitLab / Gitee | 双仓库同步、版本控制 |
| 会议与录屏 | 腾讯会议 / 钉钉会议 | Zoom / Google Meet | 周会、审片会 |
| 设计审阅 | 稿定 / Figma / 蓝湖 | Frame.io | 关键帧审阅、批注 |

---

## 二、建立检查清单

- [ ] 创建项目沟通群（含教师/导师、PM、各角色负责人）
- [ ] 创建任务看板项目，导入 [`项目进度检查清单.md`](./templates/progress_checklist.zh.md)
- [ ] 建立共享文档空间，上传项目计划书与 SOP
- [ ] 建立共享云盘文件夹，设置目录结构与命名规范
- [ ] 初始化 Git 仓库并配置双仓库同步（见 [`08_Automation/sync_repos.sh`](../08_Automation/sync_repos.sh)）
- [ ] 预约第一次师生同步会时间
- [ ] 制定会议纪律：每周例会、每日站会、紧急问题响应机制

## 四、第一次师生同步会议程

| 议程 | 时长 | 负责人 |
|------|------|--------|
| 项目背景与目标介绍 | 10 min | PM |
| 专家团队与分工确认 | 10 min | PM |
| 项目计划书与检查清单审阅 | 15 min | 教师/导师 |
| 第一周冲刺目标与任务分派 | 15 min | PM + 技术总监 |
| 风险与问题讨论 | 10 min | 全体 |
| 下一步行动与会议节奏 | 5 min | PM |
| 答疑 | 5 min | 全体 |
